# План: новая форма добавления события в календаре

## Цель

В модальном окне по тапу на день — сначала выбор из двух вариантов: **«Запись о сексе»** или **«Пожелание на сегодня»**, затем соответствующая форма. Данные сохранять в Firestore; закладка для статистики (любимая поза, прогресс по позам камасутры).

---

## 1. Первый экран модалки: выбор типа

- При открытии модального окна (добавить событие на день X) показывать **два основных действия**:
  - **«Запись о сексе»** — переход к форме записи о сексе.
  - **«Пожелание на сегодня»** — переход к форме пожелания на ближайшие 2 дня.
- Кнопка «Отмена».
- При редактировании существующего события — открывать форму того же типа без повторного выбора.

---

## 2. Модели данных

**Тип события:** одно поле в документе `events`: `kind: 'sex_record' | 'wish_today'` + опциональные поля по типу.

**Запись о сексе (sex_record):**
- `kind`, `date`, `userId`, `partnerId`
- `sexTypes`: массив строк (оральный, анальный, классический, мастурбация, совместная мастурбация, групповой секс)
- `poseIds`: массив id поз из справочника камасутры
- `toyIds` или `toyNames`: массив (игрушки — отдельный справочник пользователя)
- `durationMinutes`: int?
- `satisfactionRating`: int? (1–5)
- `note`: string?

**Пожелание на сегодня (wish_today):**
- `kind`, `date` (начало), дата окончания = date + 2 дня (хранить или вычислять)
- `userId`, `visibleToPartners: true`
- `sexTypes`: массив (те же значения)
- `contentLink?`, `contentText?`, `imageUrl?`
- Партнёры видят в календаре/приложении за эти дни

**Справочники:**
- **Позы камасутры:** статический список в коде (`lib/data/kamasutra_poses.dart`) — id + название.
- **Игрушки:** коллекция Firestore `user_toys` (или `users/{uid}/toys`) — имя, userId. В форме — выпадающий список + «добавить новую».

---

## 3. Файлы и репозитории

- `lib/models/calendar_event.dart` — расширить: `kind`, поля для sex_record и wish_today, toMap/fromFirestore.
- `lib/data/kamasutra_poses.dart` — константный список поз.
- `lib/models/user_toy.dart` + `lib/services/user_toys_repository.dart` — игрушки пользователя.
- `lib/services/events_repository.dart` — сохранение/чтение расширенной модели; при необходимости запрос событий партнёров для wish_today.
- Загрузка картинки пожелания: Firebase Storage, в модели хранить `imageUrl`.

---

## 4. UI модального окна

**Шаг 0:** Выбор «Запись о сексе» / «Пожелание на сегодня».

**Форма «Запись о сексе»:**
- Дата, партнёр (dropdown).
- Тип секса: мультиселект (чипы).
- Позы: мультиселект из списка камасутры.
- Игрушки: мультиселект + «Добавить игрушку».
- Длительность (минуты), оценка 1–5, заметка.
- Сохранить / Удалить / Отмена.

**Форма «Пожелание на сегодня»:**
- Подпись «на ближайшие 2 дня».
- Тип секса (мультиселект).
- Ссылка (URL), текст, загрузка картинки.
- «Видно партнёрам» (по умолчанию true).
- Создать / Отмена.

---

## 5. Отображение в календаре

- В списке дня показывать оба типа: кратко «Запись о сексе» / «Пожелание» + контент.
- Тап по событию — открыть форму редактирования того же типа.

---

## 6. Порядок внедрения

1. Справочник поз камасутры.
2. Модель UserToy и UserToysRepository.
3. Расширение CalendarEvent (kind + поля), обновление EventsRepository.
4. Первый экран модалки (выбор типа).
5. Форма «Запись о сексе».
6. Форма «Пожелание на сегодня» + загрузка картинки (Storage).
7. Отображение в календаре по типу.
8. Правила Firestore (и Storage при необходимости).

Статистика (любимая поза, прогресс по камасутре) — отдельно позже; данные уже закладываются (poseIds).
